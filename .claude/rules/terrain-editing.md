# Terrain Editing Patterns

When restructuring or updating Terrain.md:

- **Temporal structure for action items.** Organize tasks by *when* (today/tomorrow/pending), not by entity or topic.
- **Merge thin categories.** Don't create separate sections for 2-3 bullet lists at the same priority level -- combine them.
- **Organize by dependency, not taxonomy.** If task B is enabled by project A, put B under A.
- **Drop dead projects from the active table.** Paused/indefinite items don't belong on an operating dashboard.
- **Ask about stale project statuses.** Don't leave stale data from prior versions -- ask if unsure.
- **Never re-add removed content.** If something is missing between reads, it was removed deliberately. Don't restore it.
- **Title lives in the filename** -- `# Terrain` heading in the body. Include a timestamp on `Last updated`.
- **Temporal headers in Now.** Group actions by `**Today:**`, `**Tomorrow:**`, `**[Day] [Date]:**`, `**This week:**`. Flat checkboxes under each header -- no nested sub-items.
- **Contextual emojis for Now actions.** Use emojis that signal action type. Not decorative -- functional scanability.
- **Only update status for confirmed decisions.** Terrain reflects decisions the user has explicitly made.
- **Move tasks, don't duplicate.** When a task changes sections, delete from old and create in new.
- **Re-read before editing.** Sync tools can modify Terrain.md between reads. Always re-read immediately before editing.
- **Decision Queue section.** Lives between Inbox and Now. Simple bullet format. Items removed when resolved. Stale after 7 days.
